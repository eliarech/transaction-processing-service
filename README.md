# Transaction Processing Service

A robust, scalable microservice for handling financial transactions with MongoDB persistence.

## Overview

This service provides a RESTful API for creating, retrieving, updating, and deleting financial transactions. It uses MongoDB for data storage and is built with Spring Boot.

## Features

- Create new transactions with validation
- Retrieve transactions by ID, reference, or account ID
- Filter transactions by date range and type
- Update transaction status
- Delete transactions
- Duplicate transaction prevention
- Swagger API documentation

## Technology Stack

- Java 11
- Spring Boot 2.3.1
- Spring Data MongoDB
- Maven
- Docker & Docker Compose
- Swagger/SpringFox for API documentation
- Lombok for reducing boilerplate code

## Getting Started

### Prerequisites

- JDK 11+
- Maven 3.6+
- Docker and Docker Compose (for containerized deployment)
- MongoDB 4.4+ (if running locally)

### Running Locally

1. Clone the repository
```bash
git clone https://github.com/eliarech/transaction-processing-service.git
cd transaction-processing-service
```

2. Build the application
```bash
mvn clean package
```

3. Run the application
```bash
java -jar target/transaction-processing-service-1.0.0-SNAPSHOT.jar
```

### Using Docker

```bash
docker-compose up -d
```

## API Endpoints

### Create a New Transaction
- **URL**: `/api/transactions`
- **Method**: `POST`
- **Request Body**: Transaction data
- **Response**: Created transaction with ID
- **Status Codes**:
  - 201: Transaction created
  - 400: Invalid request data
  - 409: Transaction with same reference already exists

Example request:
```json
{
  "reference": "TRX-12345",
  "accountId": "ACC-789",
  "amount": 500.75,
  "currency": "USD",
  "type": "PAYMENT",
  "description": "Monthly subscription payment"
}
```

### Get Transaction by ID
- **URL**: `/api/transactions/{id}`
- **Method**: `GET`
- **Path Parameter**: Transaction ID
- **Response**: Transaction data
- **Status Codes**:
  - 200: Success
  - 404: Transaction not found

### Get Transaction by Reference
- **URL**: `/api/transactions/reference/{reference}`
- **Method**: `GET`
- **Path Parameter**: Transaction reference
- **Response**: Transaction data
- **Status Codes**:
  - 200: Success
  - 404: Transaction not found

### Get Transactions by Account ID
- **URL**: `/api/transactions/account/{accountId}`
- **Method**: `GET`
- **Path Parameter**: Account ID
- **Response**: List of transactions for the account
- **Status Codes**:
  - 200: Success (returns empty list if no transactions found)

### Get Transactions by Account and Date Range
- **URL**: `/api/transactions/account/{accountId}/range`
- **Method**: `GET`
- **Path Parameter**: Account ID
- **Query Parameters**:
  - `start`: Start date-time (ISO format)
  - `end`: End date-time (ISO format)
- **Response**: List of transactions within date range
- **Status Codes**:
  - 200: Success (returns empty list if no transactions found)

Example usage:
```
GET /api/transactions/account/ACC-789/range?start=2021-10-01T00:00:00Z&end=2021-10-31T23:59:59Z
```

### Update Transaction Status
- **URL**: `/api/transactions/{id}/status`
- **Method**: `PATCH`
- **Path Parameter**: Transaction ID
- **Query Parameter**: `status` (PENDING, COMPLETED, FAILED, CANCELLED)
- **Response**: Updated transaction data
- **Status Codes**:
  - 200: Success
  - 404: Transaction not found
  - 400: Invalid status value

Example usage:
```
PATCH /api/transactions/60f1a5c7e8d72a1234567890/status?status=COMPLETED
```

### Get Transactions by Type
- **URL**: `/api/transactions/type/{type}`
- **Method**: `GET`
- **Path Parameter**: Transaction type
- **Response**: List of transactions of the specified type
- **Status Codes**:
  - 200: Success (returns empty list if no transactions found)

Example usage:
```
GET /api/transactions/type/PAYMENT
```

### Delete Transaction
- **URL**: `/api/transactions/{id}`
- **Method**: `DELETE`
- **Path Parameter**: Transaction ID
- **Response**: No content
- **Status Codes**:
  - 204: Transaction successfully deleted
  - 404: Transaction not found

## Transaction Model

The main entity in this service is the Transaction, with the following properties:

| Field         | Type          | Description                                      |
|---------------|---------------|--------------------------------------------------|
| id            | String        | Unique identifier (generated by MongoDB)         |
| reference     | String        | Unique transaction reference                     |
| accountId     | String        | Account identifier                               |
| amount        | BigDecimal    | Transaction amount                               |
| currency      | String        | Currency code (e.g., USD, EUR)                   |
| type          | String        | Transaction type                                 |
| description   | String        | Optional transaction description                 |
| status        | Enum          | PENDING, COMPLETED, FAILED, or CANCELLED        |
| createdAt     | LocalDateTime | Creation timestamp                               |
| lastUpdatedAt | LocalDateTime | Last update timestamp                            |

## Environment Configuration

Key application settings can be configured in `application.properties` or via environment variables:

- `SPRING_DATA_MONGODB_HOST` - MongoDB host
- `SPRING_DATA_MONGODB_PORT` - MongoDB port
- `SPRING_DATA_MONGODB_DATABASE` - MongoDB database name

## Performance Considerations

- MongoDB connection pooling is configured for high throughput
- Indexing is used for optimized query performance
- Asynchronous processing can be implemented for high-load scenarios

## Error Handling

The service provides clear error messages for common scenarios:

- `TransactionNotFoundException`: When a transaction cannot be found
- `DuplicateTransactionException`: When attempting to create a transaction with an existing reference
